# Ecosystem

_Written during `/deliberate init`. This file is the **single source of truth** for the ecosystem roster, player details, and first-party monitoring sources used by cases and the periodic landscape brief (`/deliberate brief`). List every qualified, strategically material adjacent product, complement, channel, or mover here and nowhere else in project context; use no numeric target or cap. A complement is a distinct product or service used before, alongside, or after this product in the same end-to-end workflow where co-use or interoperability raises the value of both for shared users. It is not a substitute for this product's primary job, a technical dependency included for that reason alone, or any generic tool the same user happens to use. Adjacent products operate in other markets or niches but offer useful analogies through similar strategy, packaging, go-to-market motion, goals, technology, or product dynamics. Include a player only when current first-party evidence establishes the specific relationship or analogy, material decision relevance, and at least one useful monitoring source. Do not include vendors merely because the product depends on them, and never derive this roster from package manifests, lockfiles, or implementation inventories._
